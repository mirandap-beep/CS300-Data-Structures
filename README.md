# CS300-Data-Structures
Portfolio artifacts and coursework from CS 300: Data Structures and Algorithms
The main problem I was solving in this course was creating software that would allow academic advisors at ABC University to efficiently access and manage course information. The program needed to load course data from a file, store it using different data structures, and display courses in alphanumeric order while also allowing users to search for specific course information and prerequisites. Throughout the projects, I worked with vectors, hash tables, and binary search trees to determine which data structure provided the best balance of run-time efficiency and memory usage. Understanding data structures was important because the choice of structure directly affected how quickly data could be stored, searched, and displayed.

While working through the activities and projects, I encountered several roadblocks, including file input issues, compilation errors, and problems organizing data correctly. I overcame these challenges by testing my code incrementally, reviewing error messages, and refining my algorithms until the program worked as expected. Comparing the performance of different data structures also helped me better understand how algorithm efficiency impacts software design. This project expanded my approach to designing software by encouraging me to think about scalability, efficiency, and selecting the appropriate data structure before writing code.

My work in this course has also changed the way I write programs by emphasizing the importance of maintainability, readability, and adaptability. I learned to organize code into functions, use meaningful variable names, and create reusable structures that make programs easier to understand and modify. Developing pseudocode before implementation helped me plan solutions more effectively and reduced errors during development. Overall, this course strengthened my problem-solving skills and provided a deeper understanding of how data structures and algorithms influence software development.
